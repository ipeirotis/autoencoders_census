## Verification Gaps from 01-VERIFICATION.md

### Gap 1: Legacy Route Bypasses Authentication (Blocker)
**Location:** `frontend/server/index.ts` lines 156-173
**Issue:** Unauthenticated route `/api/jobs/job-status/:jobId` exists alongside protected route in `jobsRouter`
**Impact:** Users can check job status without authentication, violating SEC-01 requirement
**Evidence:**
```typescript
// Line 156-173 in index.ts
app.get("/api/jobs/job-status/:jobId", async (req, res) => {
  // No requireAuth middleware - authentication bypass
  try {
    const { jobId } = req.params;
    const doc = await firestore.collection("jobs").doc(jobId).get();
    // ... returns job data to unauthenticated user
  }
});
```

**Why it exists:** Added before jobs router was created (Plan 01-06). Now redundant.

### Gap 2: Inconsistent Error Logging (Warning)
**Location:** `frontend/server/routes/jobs.ts` lines 51, 80, 98
**Issue:** Using `console.error()` instead of `logger.error()` from Winston
**Impact:** Error logs missing structured context (user ID, request metadata) needed for Cloud Logging queries
**Evidence:**
```typescript
// Lines 51, 80, 98
console.error("Error generating signed URL:", error);
console.error("Error starting job:", error);
console.error("Error checking status:", error);
```

**Why it exists:** Pattern from original code before logger infrastructure (Plan 01-01).

    const { jobId } = req.params;
